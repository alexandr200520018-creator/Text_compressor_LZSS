# Text_compressor_LZSS

Приложение (консольное, на языке c++) реализует алгоритм LZSS для сжатия текстовых сообщений/файлов. Поддерживает как коды из таблицы ASCII, так и пользовательские, подгружаемые из текстовых файлов (codes.txt: символ, табуляция, код символа, перенос строки). 

При запуске программа предлагает выбрать один из трёх кодов (кастомный, дозволенные символы из которого выведутся в консоль, ASCII 7bit, ASCII 8bit), далее выводится меню: сжать сообщение, сжать текстовый файл, извлечь сжатое, выход. 

При выобе опций сжать она предлагает ввети объёмы словаря и буфера и имя файла/текстовое сообщение. В случае ошибки чтения (имеются нечитаемые символы), выводится сообщение и некоторая отладочная информация. После завершения сжатия выводится статистика сжатия (отношение входных данных к непосредственно битовому размеру полученного кода), а полученный код (в шестнадцатиричном виде) сохраняется в текстовый файл формата .lzss (в названии которого указывается дата и время проведения сжатия, во избежание ошмбки с дублированием файлов с одинаковым названием) вместе с объёмами словаря и буфера и типом кода (соответственно для декодирования кастомного кода необходимо будет иметь файл с ним).

При выборе опции извлечь, выводится список файлов формата lzss, доступных для декодирования (должны находится в папке с программой) и предлагается ввести имя нужного файла. После завершения декодирования выводится статистика, а извлечённое сообщение сохраняется в текстовый файл.
Процессы кодирования и декодирования пошагого логируются в соответствующие текстовые файлы. 

К приложению приложены файл с примером кастомного кода и текстовые файлы для тестов работы программы.

В силу ограничений системы безопасности виндовс мы не можем ввести слишком большой объём для словаря и буфера, из-за чего чаще всего мы не можем выйграть в количестве символов в сравнении с оригинальным текстом (однако по чисто битовому объёму, если считать кажный символ из кода по 4 бита, то всё успешно). В связи с этим я утверждаю, что эта программа скорее предназначена для илюстрации работы алгоритма, однако при незначительной доработке мы можем научиться записывать в результат непосредственно HEX знаки.
